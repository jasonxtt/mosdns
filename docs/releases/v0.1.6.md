# v0.1.6

## Summary

This release removes nft-related integrations from the fork and aligns the downloadable config packages accordingly.

## Removed

- removed `nft_add` support from the binary
- removed nft-related source directories and legacy repo cruft
- removed the `nft_add` rule type from the WebUI

## Config impact

This release is **not compatible** with older configs that still reference `nft_add`.

If your current config already contains `nft_add` entries, update the config before or together with the binary upgrade.

## How to upgrade

### Existing users already on the current WebUI fork

Only one config file needs to be updated:

- `sub_config/rule_set.yaml`

You can apply the incremental package from the file repo:

- `mosdns/config/config_up.zip`

This incremental package only updates `sub_config/rule_set.yaml` and does not reset user-maintained files such as:

- `upstream_overrides.json`
- `config_overrides.json`
- custom rule files under `rule/`

### Fresh deployment or full template replacement

Use the latest full package:

- `mosdns/config/config_all.zip`

After download:

1. Extract the package.
2. Rename the extracted directory to `mosdns` if needed.
3. Place it under `/cus`.
4. Final path should be `/cus/mosdns`.

## Notes

- Production users who upgrade manually should replace the binary and then apply the incremental config update.
- If you previously used nft-based rules, they must be removed from your config before starting this version.
