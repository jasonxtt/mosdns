# v0.4.13

## Summary

This patch release is the final clean correction tag for the same UI-fix
payload that was intended for `v0.4.12`, after the previous tag still inherited
the release sequencing mistake.

- keeps the compact `/log` navigation adjustments already validated on device
- keeps the maintained `/` `主题与外观` block in the restored compact top-right
  grid position
- gives GitHub cloud builds and online updates one final clean tag that points
  at the correct commit

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## UI

Highlights in this release:

- cloud-built and manually deployed artifacts now align to the same intended UI
  state
- the maintained `/` appearance module stays out of the regressed full-width
  layout
- the legacy `/log` compact navigation refinements remain included
