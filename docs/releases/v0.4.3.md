# v0.4.3

## Summary

This release extends dedicated upstream groups on the main branch and adds a DDNS-specific foreign-preferred resolution toggle in the maintained root Vue UI.

- dedicated `special_groups` now support slots `>=50` instead of stopping at `59`
- the main runtime regenerates `sub_config/special_groups.yaml` automatically from persisted group state before config load
- `DDNS` domains can now prefer the foreign upstream path without cache when `switch16` is enabled

## Added

- added `switch16` in the maintained root Vue UI for `国外解析DDNS`
- added backend-generated `special_groups.yaml` sync from persisted runtime state
- added dedicated-group config sync coverage in `coremain/api_special_groups_test.go`

## Changed

- kept the main branch dedicated-routing flow on `sequence_special_v4` / `sequence_special_v6` / `sequence_special_ot`
- saving or deleting a dedicated upstream group now rewrites the generated special-groups config and schedules a self-restart
- DDNS queries matched by `ddnslist` now use `prefer_foreign_nocache` first and fall back to `domestic` when `switch16` is on

## Config impact

This release **does** require updated main config files if you want the new behavior.

The incremental config package for this release updates:

- `config_custom.yaml`
- `sub_config/switch.yaml`
- `sub_config/special_groups.yaml`
- `rule/switch16.txt`

If a deployment is still using the temporary local-testing config that references `sequence_special_all`, restore the standard main-line `process_v4.yaml`, `process_v6.yaml`, and `process_ot.yaml` files before upgrading the binary.
