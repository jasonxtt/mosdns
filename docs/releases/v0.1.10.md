# v0.1.10

## Summary

This release applies a conservative upstream follow-up on the UDP fast path without pulling in the larger `fastCache` redesign.

## Changed

- removed the `fastCache` FNV-1a input truncation so long DNS questions no longer hash on only the first 128 bytes
- normalized `ClientAddr` with `Unmap()` before the UDP fast path runs, reducing IPv4-mapped IPv6 ambiguity in the fast path

## Config impact

This release does **not** require a config change for existing users.

No incremental config package update is needed for this release.

## Notes

- this release intentionally does **not** adopt the upstream large-memory `fastCache` layout change
- this release intentionally does **not** touch the removed nft-related path
