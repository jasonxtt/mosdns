# v0.1.7

## Summary

This release adds manual domain lists for dedicated routing groups and improves query-log visibility for rule matching and upstream selection.

## Added

- dedicated routing groups can now use two sources at the same time:
  - online `srs` rule sources
  - manual domain lists edited in `规则管理 -> 名单管理`
- after creating a dedicated group in `系统 -> 高级设置`, the group now appears automatically in list management

## Changed

- query-log details now show `匹配来源` for online diversion matches
- query-log details now show:
  - `最终上游组`
  - `最终上游`
- `最终序列` is no longer shown in the WebUI query-log detail panel
- the legacy `NFT IP` item was removed from list management

## Config impact

This release requires a config update for users who want to use manual rules in dedicated groups.

The incremental package for this release updates:

- `sub_config/special_groups.yaml`

That change adds `special_manual_50-59` and maps them into the same dedicated groups as `special_route_50-59`.

Manual rule files are stored under:

- `rule/special_50.txt`
- `rule/special_51.txt`
- ...
- `rule/special_59.txt`

These files do not need to exist before upgrade. They will be created after saving manual rules in WebUI.

## How to upgrade

### Existing users already on the current WebUI fork

Update the binary to `v0.1.7`, then apply:

- `mosdns/config/config_up.zip`

This incremental package updates only:

- `sub_config/special_groups.yaml`

It does not reset user-maintained files such as:

- `upstream_overrides.json`
- `config_overrides.json`
- custom rule files under `rule/`

### Fresh deployment or full template replacement

Use the latest full package:

- `mosdns/config/config_all.zip`

## Notes

- Dedicated-group manual rules and online rules now enter the same routing slot together.
- Dedicated-group deletion also cleans up the matching manual rule file.
