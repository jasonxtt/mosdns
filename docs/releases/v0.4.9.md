# v0.4.9

## Summary

This patch release continues the `/log` compatibility UI cleanup and finishes
the detail-view workflow that was refined through real-device testing.

- improves `/log` desktop and mobile layout consistency across primary nav,
  secondary nav, active states, glass panels, and notice bubbles
- expands overview ranking drill-down so Top domains, Top clients,
  diversion-rule stats, and slow-query entries all behave like first-class
  detail entry points
- fixes modal clipping and offset issues by moving detail dialogs out of the
  clipped `/log` page container
- adds copy and filter shortcuts to slow-query detail, with direct jump into
  live query search

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## Backend and UI notes

Highlights in this release:

- `/api/v2/audit/logs` now supports exact `domain_set` and `effective_tag`
  filters for ranking drill-down and rule-specific detail views
- ranking detail and query detail dialogs render more reliably on narrow
  screens and within the `/log` compatibility shell
- filtering from overview details now switches to `查询日志 -> 实时查询` and
  applies the chosen search value automatically
