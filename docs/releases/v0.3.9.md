# v0.3.9

## Summary

This release adds configurable WebUI port management in system settings and makes internal restart/task callbacks follow the active WebUI port instead of relying on a hardcoded `9099`.

It also includes requery status diagnostics and UI theme/interaction polish for menu and switch visibility.

## Added

- end-to-end WebUI port management:
  - new APIs: `GET /api/v1/system/webui-port` and `POST /api/v1/system/webui-port`
  - persistent settings file: `webui_port_settings.json`
  - system-settings UI module with current/target port and confirm-and-restart action
- requery status now includes `last_error` for failure diagnostics in UI

## Changed

- local restart endpoint resolution now follows active configured WebUI listen address
- update-manager/config-manager restart hooks no longer rely on fixed `http://127.0.0.1:9099/...`
- requery URL action calls normalize local loopback URLs to configured WebUI port
- data-management requery panel now displays latest failure reason
- sub-second requery completion display now shows `耗时 <1秒`
- dark theme switch ON knob contrast improved (white knob)
- first-level and second-level menu buttons now fully invert with light/dark theme
- removed extra selected outline effect on second-level menu buttons
- unified top subnav panel structure across major pages and reduced visible layout jitter during main-tab switches

## Config impact

This release does **not** require manual updates to existing `config_up.zip`.

`webui_port_settings.json` will be generated automatically in runtime config directory when saving WebUI port settings.
