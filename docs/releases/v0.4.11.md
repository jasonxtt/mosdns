# v0.4.11

## Summary

This patch release finishes another round of `/log` navigation tightening and
restores the maintained `/` system-settings appearance module after a shared-UI
regression.

- moves `刷新` / `重启` into the mobile `/log` top-row button area
- keeps desktop `/log` menus more compact and adjusts menu behavior across
  screen widths
- restores the maintained `/` `主题与外观` block to the historic compact
  top-right card inside the four-panel system-settings grid
- keeps the latest `/log` mobile UI updates without forcing the maintained `/`
  UI into the legacy full-width appearance layout

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## UI

Highlights in this release:

- legacy `/log` mobile and desktop navigation spacing is more compact and more
  stable across breakpoints
- the maintained `/` appearance settings are back in the expected compact
  system-settings card instead of rendering as a full-width block below the
  four-panel row
- `/` and `/log` appearance structures are separated again so shared styling
  changes in the compatibility UI do not misplace the maintained UI
