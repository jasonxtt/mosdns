# v0.4.6

## Summary

This release refines the maintained system settings UI and trims successor
docs for faster onboarding.

- the config update status panel now shows a user-facing config version label
  (`v1`, `v2`) instead of exposing raw schema numbers
- the config version row in system settings now matches the version badge
  styling used by binary update status
- AI-facing project docs were shortened to keep route mapping, config-release
  rules, and deployment flow easy to pick up in a fresh thread

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## UI

System settings now show:

- `当前配置版本 v2`
- the same `已是最新` badge style used in the binary version row
