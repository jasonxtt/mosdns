# v0.5.0

## Summary

This release adds dedicated listen-port support for `special_groups` and
reworks how dedicated groups are surfaced in both the maintained `/` UI and
the compatibility `/log` UI.

- a dedicated routing group can now own a separate UDP/TCP DNS listen port
- queries arriving on that dedicated port bypass the normal port-53 diversion
  chain and go directly to the group's upstream path
- the maintained `/` upstream-settings page now keeps dedicated-group details
  out of the default view and uses a compact same-row summary plus a full
  management modal
- the `/log` compatibility UI now follows the same summary and management
  pattern

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## UI

Highlights in this release:

- `/` shows dedicated groups in a compact summary next to `添加上游DNS`
- `/` and `/log` both open a full dedicated-group manager modal for edit,
  delete, and create actions
- rules management now exposes the dedicated listen-port field alongside the
  dedicated group name
