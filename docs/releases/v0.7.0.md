# v0.7.0

## Summary

This release introduces transactional binary updates with pre-download checks
and automatic rollback protection.

- every release publishes a hash-verified update manifest alongside its
  binary assets
- MosDNS downloads and verifies the target binary before changing the running
  installation
- a config package is downloaded only when the target release requires a
  newer config schema than the currently applied schema
- the existing binary is retained as the temporary update guard
- the guard starts and probes the candidate version before handing over the
  service
- candidate startup failures restore both the previous binary and the config
  backup

## Compatibility boundary

`v0.7.0` keeps config schema `3` and package ID `main-config-schema-3`, so this
release itself is a binary-only update.

The transactional update protocol applies to updates performed by `v0.7.0`
and later. Older binaries still use their historical updater when installing
`v0.7.0`.

## Release metadata

The release workflow publishes `mosdns-update-manifest.json`. Future
structural config releases must update `scripts/release-update-metadata.json` with the
new schema, package ID, immutable config package URL, and SHA-256 before the
release tag is pushed.
