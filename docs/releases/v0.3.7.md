# v0.3.7

## Summary

This release hardens the `刷新分流缓存` execution flow to avoid occasional false `0秒完成` feedback after manual trigger, and keeps task-state feedback stable in the Vue dashboard.

## Changed

- requery backend robustness:
  - unified manual trigger and scheduler paths with atomic task-start state initialization
  - persisted `running` state before goroutine execution to eliminate trigger/status race windows
- data-management frontend robustness:
  - added short-lived trigger-pending state after manual task start
  - switched to adaptive polling cadence (`1s` pending, `5s` running)
  - improved duration text for sub-second tasks (`耗时 <1秒`)
- panel transparency consistency:
  - extended panel background transparency/glass behavior to requery scheduler surfaces
  - fixed panel-background button text contrast token behavior for dark themes

## Config impact

This release does **not** require a config change for existing users.

No incremental config package update is needed for this release.
