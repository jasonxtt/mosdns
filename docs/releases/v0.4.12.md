# v0.4.12

## Summary

This patch release republishes the latest UI fixes as a clean follow-up binary
because the previous `v0.4.11` tag pointed at the wrong commit for GitHub
cloud builds and online-update consumers.

- keeps the compact `/log` navigation adjustments already validated on device
- ensures the maintained `/` system-settings `主题与外观` block is shipped in
  the restored compact top-right grid position
- gives GitHub cloud builds and online updates a clean release tag that points
  at the correct UI-fix commit

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## UI

Highlights in this release:

- production and cloud-built artifacts now match the already validated test
  host UI state
- the maintained `/` appearance module is no longer regressed to the
  full-width layout in release binaries
- the legacy `/log` compact navigation refinements remain included
