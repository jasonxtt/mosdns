# v0.3.12

## Summary

This release improves query-log operator workflows in two places:

- the Vue query-detail dialog now mirrors legacy `/log` with inline `复制 / 筛选` actions
- audit `生效标签` now reflects the final routing result more clearly when a direct candidate is ultimately sent to proxy, or when learned memory is reversed by the final path

## Added

- added inline quick actions in Vue `查询日志 -> 查询详情`:
  - `客户端`
  - `域名`
  - `分流规则`
  - `Trace ID`
- each of the fields above now supports:
  - `复制`
  - `筛选`

## Changed

- adjusted audit effective-tag display:
  - first-hit direct candidates that finally go through `sequence_fakeip_addlist` now display `直连候选转代理`
  - learned-route reversals are shown explicitly as `记忆直连转代理` or `记忆代理转直连`
  - learned stable proxy results still display `记忆代理`

## Config impact

This release does **not** require manual `config_up.zip` updates.
No YAML config migration is required for existing deployments.
