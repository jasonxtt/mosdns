# v0.4.10

## Summary

This patch release finishes another round of `/log` compatibility UI fixes
after more real-device validation across desktop and mobile browsers.

- keeps primary nav, secondary nav, and selected-state styling more consistent
  across light and dark themes in `/log`
- restores panel-background transparency and blur behavior in non-overview
  `/log` modules such as rules, upstream settings, and system settings
- fixes rule-management editor dialogs that were still clipped by the parent
  panel container
- keeps rule-editor actions visible by making the modal body scroll correctly
  and aligning the footer actions

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## UI

Highlights in this release:

- legacy `/log` glassmorphism settings now apply more consistently outside the
  Overview page
- diversion-rule, AdGuard-rule, and special-group editor dialogs render at the
  `body` level so they no longer get clipped by the surrounding page panel
- rule-management action buttons use a more consistent size and spacing
