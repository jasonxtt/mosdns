# v0.1.13

## Summary

This release fixes two WebUI behavior issues in the merged dashboard: diversion-type label mapping for `cuscn/cusnocn`, and inline confirmation popover overflow near the bottom of the screen.

## Changed

- corrected diversion-type labels in WebUI:
  - `cuscn` is now displayed as `!cn@cn`
  - `cusnocn` is now displayed as `cn@!cn`
- updated diversion-rule help text so type descriptions match the actual backend routing semantics
- improved inline confirmation popover placement:
  - popovers now flip upward when the trigger is near the viewport bottom
  - confirmation actions remain visible and clickable at the bottom of long pages

## Config impact

This release does **not** require a config change for existing users.

No incremental config package update is needed for this release.

## Notes

- this release is a source-side WebUI bugfix release
- backend diversion logic is unchanged; only UI labeling and popover positioning were adjusted
