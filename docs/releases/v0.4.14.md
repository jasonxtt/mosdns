# v0.4.14

## Summary

This patch release restores the maintained root `/` data-management workflow
to its earlier behavior after a recent UI regression hid two of the three
expected panels.

- the maintained `/` data-management page again shows `缓存管理`,
  `域名列表统计`, and `刷新分流缓存` together by default
- the compatibility `/log` UI is intentionally left unchanged
- embedded frontend assets and the release binary are aligned again so the
  shipped UI matches the intended maintained workflow

## Config update behavior

This is a binary-only release.

- required config schema remains `2`
- required config package ID remains `main-config-schema-2`
- no new `config_up.zip` is required for this release

## UI

Highlights in this release:

- the maintained `/` data-management page no longer regresses to a
  cache-only layout
- operators can again reach domain-list statistics and requery controls
  directly from the default maintained UI flow
- the legacy `/log` compatibility surface stays untouched
