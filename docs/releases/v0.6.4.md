# v0.6.4

## Summary

`v0.6.4` refines SOCKS behavior for the foreign upstream groups in the
maintained WebUI and keeps historical configs compatible.

- `foreign` and `foreignecs` now have a `使用 socks 代理` toggle in upstream
  settings
- when the toggle is off, the upstream connects directly and does not inherit
  the global SOCKS value
- when the toggle is on and the per-upstream SOCKS value is empty, both groups
  continue inheriting the global SOCKS value
- historical empty SOCKS rows for `foreign` / `foreignecs` are inferred as
  enabled, preserving previous behavior after upgrade

## Upgrade Notes

- this is a binary-only release
- no `config_up` / `config_all` update is required
- existing deployments can update only the binary
