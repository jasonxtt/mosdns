# 相对上游的改动说明

上游基础：

- `yyysuo/mosdns`

当前这个 fork 已经落地的改动，可以按下面几类理解。

## 一、WebUI 新能力

### 1. 专属分流组

新增 API：

- `GET /api/v1/special-groups/`
- `POST /api/v1/special-groups/`
- `DELETE /api/v1/special-groups/{slot}`

作用：

- 在 WebUI 中创建一个“专属分流组”
- 每个组自动绑定 `mark50-mark59` 之一
- 同时映射到：
  - 专属在线规则插件
  - 专属上游插件
  - 专属缓存

### 2. WebUI 高级设置联动

新增能力：

- 在高级设置中创建分流组
- 在高级设置中编辑专属上游
- 在规则管理中将规则直接归入某个专属分流组

### 3. 查询日志展示优化

原始内部标识仍然保留为稳定 tag/mark 逻辑，
但在前端显示时会映射为：

- `组名`
- `mark50`

避免用户只看到 `特殊上游50` 这类不直观的名称。

## 二、规则管理增强

### 1. 在线分流规则保存后自动下载

涉及插件：

- `sd_set_light`
- `sd_set`
- `si_set`

行为变化：

- 原来：保存规则源后，只写配置，不主动下载
- 现在：保存成功后，如果带 `url + files`，会自动触发下载并 reload

效果：

- 新增规则后，`.srs` 会自动落盘
- 不需要再手工点“更新”

### 2. 特殊组类型动态出现在规则管理中

原来规则类型是固定的。

现在规则类型由两部分组成：

- 内建类型
- 当前专属分流组

因此可以做到：

- 新增一个分流组
- 规则管理里立刻可选

## 三、上游管理增强

### 1. 上游配置热重载

涉及插件：

- `aliapi`

行为变化：

- 原来修改上游配置后需要重启服务
- 现在保存上游组后会对目标插件做热重载

效果：

- WebUI 改上游后即时生效
- 减少对运行中 DNS 服务的中断

## 四、配置层配套

在配置层增加了专属分流槽位体系：

- `special_50` 到 `special_59`

每个槽位包含：

- 一个分流规则集插件
- 一个专属缓存
- 一个专属上游
- 一个专属查询序列

这部分不是通用上游功能，而是为了让 WebUI 的“专属分流组”能直接落到真实可运行的配置结构。

## 五、当前值得保留的发布卖点

如果准备对外发布，最值得强调的是下面四点：

1. WebUI 可以创建专属分流组
2. 分流规则和上游组可以直接联动
3. 新增在线规则后自动下载，无需手工更新
4. 上游保存后即时生效，无需重启 mosdns

## 六、建议写清楚的边界

这个 fork 不是通用版“官方增强”，而是偏向实际使用场景驱动的增强分支，重点解决：

- 分流规则长期维护
- 上游管理在线化
- 家庭网络和代理分流的运维效率

因此在发布说明里建议明确：

- 基于 `yyysuo/mosdns`
- 与上游保持兼容思路
- WebUI 和特定配置工作流增强是当前重点
