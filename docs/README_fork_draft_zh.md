# README 草案

## 项目简介

这是一个基于 `yyysuo/mosdns` 的增强版 `mosdns` fork，重点补充了 WebUI 对“专属分流组 / 专属上游 / 在线规则自动下载 / 上游热重载”的支持。

这个分支面向的是“长期在线维护 DNS 分流规则”的使用场景，而不是只做一次性配置导入。

## 上游来源

- 上游项目：`yyysuo/mosdns`

本项目保留上游整体结构与主要配置思路，当前改动主要集中在：

- WebUI 交互能力增强
- 分流规则与上游组联动
- 在线规则管理体验
- 上游修改后即时生效

## 当前增强点

### 1. 专属分流组

- 可在 WebUI 中新增“专属分流组”
- 当前实现最多支持 10 个专属分流组
- 每个分流组都可以绑定自己的：
  - 在线规则
  - 上游组
  - 缓存

### 2. 规则与上游联动

- 在规则管理里选择某个专属分流组
- 命中该组的域名会直接走对应的专属上游

### 3. 在线规则保存后自动下载

- 新增在线分流规则后自动下载 `.srs`
- 不再需要手动点“更新”

### 4. 上游热重载

- WebUI 保存上游组后即时生效
- 不再需要重启 `mosdns`

### 5. 查询日志显示优化

- 日志中会按当前组名显示专属分流组
- 同时展示匹配来源、最终上游组和最终上游，便于排查

## 适用场景

- 家庭网络分流
- 旁路 DNS
- 代理环境下的域名分流维护
- 需要通过 WebUI 长期维护 geosite / srs 规则的场景

## 设计目标

这个 fork 解决的是下面几个实际问题：

1. 规则和上游之间缺少直接联动
2. 保存在线规则后还需要手工更新下载
3. 修改上游配置后需要重启服务
4. 查询日志里的内部标识不够直观

## 与上游的差异

本 fork 当前已实现：

- 新增专属分流组 API
- WebUI 动态分流组管理
- 上游配置热重载
- 在线规则自动下载
- 查询日志中的专属分流组友好展示
