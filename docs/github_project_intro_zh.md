# 项目简介草案

## 项目名称建议

- `mosdns-webui-plus`
- `mosdns-special-upstream`
- `mosdns-ui-enhanced`

如果你希望保持和上游更接近的命名，建议：

- `mosdns-fork-webui`

## GitHub 仓库简介

基于 `yyysuo/mosdns` 的增强版 `mosdns`，补充了面向家庭分流场景的 WebUI 能力，包括专属分流组、专属上游管理、规则保存后自动下载、上游热重载等。

## 项目定位

这个 fork 的目标不是重写 `mosdns`，而是在保留上游整体架构和配置能力的前提下，补齐一套更适合“长期在线维护分流规则”的 WebUI 工作流。

适用场景：

- 需要长期维护大量在线分流规则
- 需要把“域名分流”和“上游组”绑定起来
- 希望通过 WebUI 管理专属上游，而不是频繁手改 YAML
- 希望修改上游后即时生效，不重启服务

## 当前增强点

### 1. 专属分流组

- 新增 WebUI “专属分流组”能力
- 支持在高级设置中创建分流组
- 当前实现最多支持 10 个专属分流组
- 每个分流组都可绑定：
  - 独立在线分流规则集
  - 独立上游组
  - 独立缓存

### 2. 上游组与分流联动

- 可在高级设置中维护专属上游
- 可在规则管理中直接选择某个专属分流组
- 命中该组的域名会优先走绑定的专属上游

### 3. 上游配置热重载

- 保存上游配置后立即生效
- 不再要求手动重启 `mosdns`
- 适用于 `aliapi` 类上游组

### 4. 在线分流规则自动下载

- 新增在线分流规则后
- 后端自动下载对应的 `.srs`
- 不再要求手动点击“更新”

### 5. 查询日志显示优化

- 特殊分流规则不再只显示 `特殊上游50`
- 支持按当前分流组名称显示
- 同时补充最终序列和最终上游，便于排查

### 6. WebUI 规则管理增强

- 规则类型支持动态注入专属分流组
- 下拉项直接显示当前分流组名称
- 更适合长期维护多个专属分流入口

## 与上游的关系说明

本项目基于：

- 上游仓库：`yyysuo/mosdns`

本 fork 主要增加：

- WebUI 的专属分流组管理
- 上游配置热重载
- 在线规则自动下载
- 查询日志显示优化
- 若干适配专属分流工作流的 API

## 适合写进 README 开头的说明

这是一个基于 `yyysuo/mosdns` 的增强版分支，重点补充了 WebUI 对“专属分流组 / 专属上游 / 在线规则自动下载 / 上游热重载”的支持，适合家庭网络、代理分流、旁路 DNS 等需要长期维护规则的场景。
