# mosdns 分流顺序表（10.0.0.3）

## 说明

本文基于 `10.0.0.3` 当前运行配置整理，主要依据：

- `/cus/mosdns/config_custom.yaml`
- `/cus/mosdns/sub_config/rule_set.yaml`
- `/cus/mosdns/sub_config/process_v4.yaml`
- `/cus/mosdns/sub_config/process_v6.yaml`
- `/cus/mosdns/sub_config/process_ot.yaml`
- `/cus/mosdns/sub_config/special_groups.yaml`

这套配置不是“命中第一个域名集就立刻停”。

- 很多规则会先一起打上 `fast_mark`
- 后续再由 `sequence_ipv4` / `sequence_ipv6` / `sequence_other` 决定最终走向
- 只有少数规则是早停型，例如 `重定向`、`黑名单`、`DDNS`、`专属分流组`

## 一、A / AAAA 主分流优先级

适用于绝大多数常见域名解析请求。

| 顺位 | 规则/UI名称 | 运行标签/mark | 实际结果 |
| --- | --- | --- | --- |
| 1 | 重定向 | 直接走 `$rewrite` | 最早执行，命中后直接结束，不再看后面任何名单 |
| 2 | 黑名单 | `fast_mark 1` | 直接拦截 |
| 3 | 广告屏蔽 | `fast_mark 5` | 直接拦截 |
| 4 | DDNS 域名 | `fast_mark 6` | 直接走国内上游 `$domestic`，然后结束 |
| 5 | 专属分流组 | `mark 50-59` | 进入对应 `sequence_special_50~59`，走绑定上游并结束 |
| 6 | `!CN fakeip filter` | `!CN fakeip filter / fast_mark 9` | 直接走国外 realip 路径并结束 |
| 7 | 记忆代理 | `记忆代理 / fast_mark 12` | 直接走 `nocnfake`，优先于普通白/灰/订阅规则 |
| 8 | 记忆直连 | `记忆直连 / fast_mark 11` | 直接走直连路径，优先于普通白/灰/订阅规则 |
| 9 | 灰名单 | `灰名单 / fast_mark 7` | 进入代理候选分支 |
| 10 | `cn@!cn` | `订阅代理补充 / fast_mark 15` | 进入代理候选分支 |
| 11 | 白名单 | `白名单 / fast_mark 8` | 进入直连候选分支 |
| 12 | `!cn@cn` | `订阅直连补充 / fast_mark 13` | 进入直连候选分支 |
| 13 | `CN fakeip filter` | `CN fakeip filter / fast_mark 10` | 进入直连候选分支 |
| 14 | 中国域名 `cn` | `订阅直连 / fast_mark 16` | 进入直连候选分支 |
| 15 | 非中国域名 `!cn` | `订阅代理 / fast_mark 14` | 前面都没截走时，进入代理分支 |
| 16 | 列表外域名 | 无 | 靠上游返回 IP 再判断国内/国外 |

## 二、手动可维护规则的实际强弱

这是更接近 WebUI 使用习惯的一张表。

| 层级 | 规则 | 说明 |
| --- | --- | --- |
| 最高 | 重定向 | 命中后直接改写并结束 |
| 很高 | 黑名单 | 直接拦截 |
| 很高 | 广告屏蔽 | 直接拦截 |
| 很高 | DDNS 域名 | 直接国内解析并结束 |
| 很高 | 专属分流组 | 直接走绑定上游，优先于普通白/灰/订阅 |
| 高 | !CN fakeip filter | 直接国外 realip |
| 高 | 记忆代理 / 记忆直连 | 比普通白/灰/订阅更强 |
| 中高 | 灰名单 / `cn@!cn` | 代理候选 |
| 中 | 白名单 / `!cn@cn` / CN fakeip filter / 中国域名 `cn` | 直连候选 |
| 中低 | 非中国域名 `!cn` | 普通代理订阅 |
| 最低 | 列表外域名 | 靠探测 IP 再分流 |

## 三、订阅规则和 UI 类型对应关系

| UI显示/类型 | 插件Tag | 运行标签 | fast_mark | 归类 |
| --- | --- | --- | --- | --- |
| 中国域名 `cn` | `geosite_cn` | 订阅直连 | 16 | 直连候选 |
| 非中国域名 `!cn` | `geosite_no_cn` | 订阅代理 | 14 | 普通代理 |
| `!cn@cn` | `cuscn` | 订阅直连补充 | 13 | 直连候选 |
| `cn@!cn` | `cusnocn` | 订阅代理补充 | 15 | 代理候选 |
| 中国 IP | `geoip_cn` | 无单独 domain tag | 无 | 主要用于列表外 IP 判断，不属于域名规则主标签 |

说明：

- `中国域名` 和 `非中国域名` 会真实生效，只是运行时显示成 `订阅直连`、`订阅代理`
- 你在 WebUI 里新增订阅规则并选择这两个类型，保存后会真实参与分流
- `中国 IP` 也会生效，但它主要参与“上游返回 IP 后再判断国内/国外”的流程，不像前面几种那样直接显示成常规 `domain_set` 标签

## 四、常见冲突时谁赢

| 冲突情况 | 最终谁赢 |
| --- | --- |
| 重定向 和任何别的规则同时命中 | 重定向 |
| 黑名单 和 白名单/灰名单/订阅 同时命中 | 黑名单 |
| 专属分流组 和 白名单/灰名单/订阅 同时命中 | 专属分流组 |
| !CN fakeip filter 和 白名单/灰名单/订阅 同时命中 | !CN fakeip filter |
| 灰名单 和 白名单 同时命中 | 灰名单 |
| `cn@!cn` 和 `!cn@cn` 同时命中 | `cn@!cn` |
| 白名单 和 `!cn` 同时命中 | 白名单 |
| 中国域名 `cn` 和 非中国域名 `!cn` 同时命中 | `cn` |

## 五、非 A / AAAA 请求顺序

非 `A`、`AAAA` 请求走的是 `sequence_other`，顺序明显更简单。

| 顺位 | 规则 | 实际结果 |
| --- | --- | --- |
| 1 | 重定向 | 直接结束 |
| 2 | 黑名单 | 直接拦截 |
| 3 | 广告屏蔽 | 直接拦截 |
| 4 | DDNS 域名 | 直接国内上游 |
| 5 | 专属分流组 | 直接走专属上游 |
| 6 | 中国域名 `cn` | 走国内上游 |
| 7 | 其他所有 | 走国外上游 |

这意味着在非 `A/AAAA` 请求里，下面这些规则并不直接决定最终方向：

- 白名单
- 灰名单
- `!cn@cn`
- `cn@!cn`
- `CN fakeip filter`
- `!CN fakeip filter`

## 六、为什么日志里会同时出现多个标签

因为当前 `domain_mapper` 允许同一域名同时命中多个规则，日志里的 `命中标签` 是拼接结果，不代表“排在前面的那个一定最终生效”。

当前拼接顺序大致是：

1. 黑名单
2. 记忆无V4
3. 记忆无V6
4. 广告屏蔽
5. DDNS域名
6. 灰名单
7. 白名单
8. !CN fakeip filter
9. CN fakeip filter
10. 记忆直连
11. 记忆代理
12. 订阅直连补充
13. 订阅代理
14. 订阅代理补充
15. 订阅直连（中国域名 `cn`）

所以像：

- `记忆代理|订阅直连`
- `白名单|订阅代理`

都不稀奇，真正要看谁最终生效，还得结合：

- `final_sequence`
- `final_upstream`
- `effective_tag`

## 七、最简短结论

如果只想记住最核心的顺序，可以记成：

1. 重定向
2. 黑名单 / 广告 / DDNS
3. 专属分流组
4. `!CN fakeip filter`
5. 记忆代理 / 记忆直连
6. 灰名单 / `cn@!cn`
7. 白名单 / `!cn@cn` / `CN fakeip filter` / 中国域名 `cn`
8. 非中国域名 `!cn`
9. 列表外域名
